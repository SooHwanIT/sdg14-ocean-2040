# Ocean 2040 — 개선 내역 (대상 도전판)

기존: 단일 `App.jsx`(364줄), DOM·이모지 기반의 버튼 클릭 낚시.
개선: **실제 Canvas 게임 엔진**을 갖춘 모듈형 React 앱. 컨셉·0.1% 엔딩·업그레이드→파괴의 아이러니는 그대로, 그러나 “진짜 게임”으로.

## 2. 기술 하이라이트

### 캔버스 게임 엔진 (`src/lib/engine.js`)
- `requestAnimationFrame` 루프에 **delta-time** 적용 → 프레임레이트 독립적 움직임. 탭 비활성 복귀 시 `dt`를 클램프해 점프 방지.
- **DPR 대응 반응형**: `ResizeObserver`로 컨테이너 크기를 추적, `devicePixelRatio`(최대 2)로 백버퍼를 스케일 → 모바일/레티나에서도 선명, 데스크톱 와이드까지 자동 리레이아웃.
- **페이즈 상태기계**: `idle → casting → waiting → timing → reeling → ending`. 렌더 루프(명령형)와 React 상태(선언형)를 **완전 분리**.
- React ↔ 엔진 통신은 두 갈래로 단순화: 엔진이 매 프레임 `getState()`로 오염도·장비를 **끌어오고(pull)**, 이벤트(`onEvent`)로 결과를 **밀어낸다(push)**. 상호 리렌더 오염 없음.

### 타이밍/스킬 메커닉 (챔질 게이지)
- 입질 순간 마커가 좌우로 왕복. **초록(perfect ×1.6) / 노랑(good ×1.0) / 그 외(놓침)** 판정.
- 난이도 다이내믹: 오염도·장비 등급이 오를수록 마커가 빨라지고 안전구간이 좁아진다 → “바다가 격해질수록 손맛도 거칠어진다”는 체감.
- 반응 제한시간(3.4초) 초과 시 자동 놓침 → 버튼이 아니라 **반응 게임**.

### 절차적 렌더링 (`scene.js`, `entities.js`) — 이미지 파일 0개
- 물: 오염도→HSL 보간으로 물빛/탁도/신광/파도 수면/해저 실루엣을 매 프레임 생성.
- 물고기: 베지어 몸통 + 흔들리는 꼬리 + 등지느러미 + 눈을 path로 그리고, `grotesque` 파라미터(오염도 유도)로 **색이 병들고 종양처럼 융기하며 비닐이 걸린다**.
- 쓰레기(봉지/병/타이어/캔), 기포, 물보라 파티클, 낚싯바늘까지 전부 코드로.
- 깨끗한 물고기: `shadowBlur`로 발광 처리, 엔딩에서 전용 캔버스가 맑아지는 물·빛기둥·상승 기포와 함께 연출.

### 견고성
- `storage.js`: 스키마 버전 불일치·JSON 파손·프라이빗 모드 차단 시 **조용히 기본값**으로 폴백.
- `audio.js`: AudioContext 생성/재생 실패가 게임을 멈추지 않도록 전 구간 try/catch, 자동재생 정책 대응(`resume`).

## 3. 심사 어필 포인트

**기술**
- 리액트 안에서 “게임 루프를 어떻게 분리하는가”라는 실전 난제를 pull/push 경계로 해결한 아키텍처.
- 외부 에셋·라이브러리·API 키 0개로 물리적 움직임과 회화적 바다를 전부 절차 생성.

**UX**
- 반응형 캔버스(모바일 탭 + 데스크톱), 시작/HUD/엔딩 화면 전환, 로딩 상태, WebAudio, reduced-motion 존중까지 완성형 제품 감각.
- 버튼 낚시 → 손맛 있는 타이밍 게임으로 승격.

**임팩트 / 메시지**
- 게임의 **메커니즘 자체가 논증**이다: 오염된 물고기를 팔아 더 큰 장비를 살수록 바다가 눈앞에서 비어간다.
- 0.1%는 통계가 아니라 **수사적 장치**임을 “왜 이 게임인가” 패널에서 정직하게 밝힌다(과장 수치 배제, 널리 보고된 우려로 프레이밍).
- 엔딩의 문장 — “돈은 늘었고 바다는 줄었다” — 이 시네마틱 연출과 만나 감정적으로 착지한다.
